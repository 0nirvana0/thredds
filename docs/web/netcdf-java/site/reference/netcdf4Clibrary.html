Netcdf4 C Library Loading

Since version 4.3.13

In order to write netCDF-4 files, you must have the netCDF-4 C library, version 4.2.1 or above, compiled and available on your system, along with all supporting libraries (hdf5 etc). The details of this differ for each operating system, and our experience (so far) is documented below.

The libraries must be in a directory that the operating system knows to look in, and the CDM needs to know the netcdf-4 library name.

  • For TDS users, we recommend setting the library path and name in threddsConfig.xml.
  • For stand-alone CDM library use, you can set the library path and name in the RuntimeConfiguration file, or call ucar.nc2.jni.netcdf.Nc4Iosp.setLibraryAndPath().

We strongly recommend using 64-bit Java. We also recommend using Java 7 (Java 6 is acceptable for CDM version 4.3). In any case, use the lastest update for Java, because security improvements are constantly being made.

Windows

  1. We strongly recommend using the 64-bit Windows version and a 64-bit JVM (Java Virtual Machine).
  2. Get the pre-built windows binaries from here. (Power users are welcome to build them - but Java needs DLLs to load). You need HDF5, but not DAP, so the nc4_64_dll.zip is likely what you want.
  3. Unzip this file, and put everything from the bin and deps directories into some standard directory accesible to anyone who wants to use the netCDF-C library.
  4. The name of this directory is the library path. The library name in this case is netcdf-7 (not libnetcdf-7.dll).
  5. Pass the library name and path to the CDM library.
